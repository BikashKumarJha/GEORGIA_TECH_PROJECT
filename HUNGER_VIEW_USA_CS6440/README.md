**Hunger View USA** is a web-based dashboard designed to help visualize food insecurity across the U.S. Using an interactive map and real-time filtering, users can explore trends in food access by state or county and see how key indicators vary by region.

**Live App:** https://github.gatech.edu/pages/aalhaddi3/hunger-view-usa/

## ğŸ“¦ What It Does

- Zoomable map with automatic switching between state and county boundaries  
- Color-coded heatmap based on food desert scores  
- Filter dropdowns for region and state  
- Sortable data table with key stats (population, SNAP households, vehicle access)  
- Popups that display food insecurity info when you click on areas of the map

## ğŸ§­ User Guide

**Hunger View USA** is designed to be simple, yet informative. Hereâ€™s how to explore it:

### ğŸ—ºï¸ Map View (Top of Page)
- **State-Level View (Default Zoom)**:  
  - Each state is shaded by its *average food desert score*.
  - Click on a state to see a popup with its name and average score.
- **County-Level View (Zoomed In)**:  
  - When zoomed in far enough, the map switches to counties.
  - Counties are colored by their *individual food desert scores*.
  - Click a county to see its name, state, and score in a popup.

### ğŸ” Filters (Middle of Page)
- Use the **Region** dropdown to filter by U.S. region.
- Use the **State** dropdown to narrow the view to a specific state.

### ğŸ“Š Data Table (Bottom of Page)
- View detailed stats for each county:
  - State, County, Population, Food Desert Score, SNAP Households, No Vehicle Households
- Click any **column header** to sort the data.

## ğŸ› ï¸ Getting Started

### 1. Clone the repository

```bash
git clone https://github.gatech.edu/aalhaddi3/hunger-view-usa.git
cd hunger-view-usa
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up and run the ETL script

Before launching the app, run the ETL script to generate the required dataset.

```bash
cd etl
python data.py
cd ..
```

### 4. Start the development server

```bash
npm run dev
```

The app will run at: [http://localhost:5173](http://localhost:5173)
